# v1.0.1 Windows 启动脚本实时日志增强

## 变更背景

`v1.0.0` 的 `start-dev.ps1` 能完成双服务启动与清理，但默认在后台子进程模式下运行，终端中不直接回显 backend/website 的实时日志，不利于快速定位启动失败或端口冲突问题。

## 变更内容

- 增强 `start-dev.ps1` 为实时日志模式：
  - 使用 `.NET Process` + 标准输出/错误输出重定向。
  - 在当前终端回显两个服务日志，并附加来源前缀（`[backend]` / `[website]`）。
  - 错误流使用 `[xxx][ERR]` 标记，便于快速识别异常信息。
- 保持原有进程管理语义：
  - 任一服务退出时，统一触发清理。
  - Ctrl+C/终止时清理所有子进程，避免残留。
- 更新 `README.md`：
  - 明确 `dev:win` 的“同终端实时日志”行为说明。

## 使用方式

```powershell
npm run dev:win
```

## 预期输出示例

```text
backend pid: 12345
website pid: 23456
dev services started with live logs, press Ctrl+C to stop.
[backend] ...
[website] ...
```
