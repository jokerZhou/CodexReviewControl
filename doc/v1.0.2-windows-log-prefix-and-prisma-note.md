# v1.0.2 Windows 日志前缀修复与 Prisma 说明补充

## 变更背景

在 `v1.0.1` 实时日志模式下，终端日志前缀显示为空（`[]`），不便于区分 backend/website 输出来源。同时，Windows 首次启动时可能遇到 `workspace-client` 未生成导致后端启动失败，需要补充明确指引。

## 变更内容

- 修复 `start-dev.ps1` 日志前缀：
  - 使用 `Register-ObjectEvent -MessageData` 显式传递服务名。
  - 日志前缀恢复为 `[backend]` / `[website]`。
  - 错误日志前缀恢复为 `[backend][ERR]` / `[website][ERR]`。
- 更新 `README.md`：
  - 新增 Prisma 缺失场景的处理说明。
  - 当提示缺少 `.prisma/workspace-client` 时，执行 `pnpm run db:generate` 生成客户端。

## 使用建议

首次拉取项目后建议在 `backend` 目录先执行：

```powershell
pnpm run db:generate
```

随后回到项目根目录执行：

```powershell
npm run dev:win
```
