# v1.0.0 Windows 启动脚本支持

## 变更背景

原有根目录脚本 `start-dev.sh` 依赖 Bash 环境，在 Windows 默认 PowerShell 环境下不够直接易用。为降低本地开发启动门槛，新增 Windows 原生启动入口。

## 变更内容

- 新增 `start-dev.ps1`：
  - 并行启动 `backend` 与 `website` 的 `pnpm run dev`。
  - 任一子进程退出时，自动结束并清理另一子进程。
  - 支持 Ctrl+C/终止场景下统一清理，避免遗留后台开发进程。
- 新增 npm 脚本 `dev:win`：
  - 命令为 `powershell -ExecutionPolicy Bypass -File ./start-dev.ps1`。
- 更新 `README.md`：
  - 增加 Windows PowerShell 启动示例与脚本列表说明。

## 使用说明

在项目根目录执行：

```powershell
npm run dev:win
```

## 兼容性说明

- 适用系统：Windows（PowerShell）。
- 依赖要求：本机可执行 `pnpm.cmd`，且已完成依赖安装。
